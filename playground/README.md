# Silicon Playground

Browser playground for the **Silicon** programming language. The Silicon
compiler (the TypeScript pipeline under `../compiler`) runs **fully
client-side** — the static build inlines it into a single self-contained
`index.html`, so there is no server and no `/compile` API. The page parses,
type-checks, lowers to WAT and assembles to WASM (via `wabt`) entirely in the
browser, then instantiates and runs the module.

## Build (static, for deployment)

```sh
bun install            # from the monorepo root
bun run --cwd playground build
```

This regenerates the inlined compiler assets and emits `playground/dist/index.html`
— the whole playground in one file (CodeMirror still loads from its CDN).

Verify the in-browser compiler end-to-end (headless: compiles a program and
instantiates the emitted WASM):

```sh
bun run --cwd playground verify
```

## Local dev (optional server)

`playground/server.ts` is a Bun HTTP host that serves the UI and compiles via a
`POST /compile` endpoint — handy for iterating on the UI without rebundling:

```sh
bun run --cwd playground start     # http://localhost:3001/  (PORT=8080 to rebind)
```

The static build and the dev server share the same UI (`playground/index.html`);
the build rewrites its `fetch('/compile')` into a direct in-browser compile call
and inlines `web-env.js`.

## How the client-side compiler works

The compiler reads its assets (Ohm grammar, strata, `std.wat`, built-in
modules, the web platform) from the filesystem under Bun/Node. For the browser
there is no filesystem, so:

- `compiler/scripts/gen-web-assets.ts` inlines every asset into
  `compiler/src/assets.generated.ts` (git-ignored, regenerated by `build`).
- Each filesystem read is isolated in a `*Source.ts` module with a
  `*Source.browser.ts` variant that returns the inlined copy.
- `web/build.ts` bundles `web/entry.ts` for the browser with Bun, aliasing the
  `*Source` modules to their `.browser` variants. A tripwire fails the build if
  any Node builtin (`fs`/`path`/`url`/…) reaches the bundle.

`web/entry.ts` exposes `globalThis.SiliconCompiler.compile({ source, platform,
features })`, returning `{ success, wat, wasm (base64), exports, platform,
features }` (or `{ success: false, error }`) — the same shape the old
`/compile` endpoint produced.

## Repository layout

```
playground/        UI (index.html, web-env.js) + legacy dev server (server.ts)
web/               entry.ts (browser compile API), build.ts, verify.ts
dist/              static build output (git-ignored)
../compiler        the Silicon compiler pipeline this bundles
```

## Deploy (Vercel)

This directory deploys as its own Vercel project (`vercel.json`):
`buildCommand: bun run build`, `outputDirectory: dist`, framework `null`. Set
the project's **Root Directory** to `playground` and point the desired
subdomain at it.
