Sigil 1.1 — User Stories

Work consciously deferred out of 1.0 scope. Phase numbers are retained from v1-user-stories.html so cross-references stay valid; nothing here gates the 1.0 release.

Scope philosophy: 1.0 ships when the four pillars (language, compiler, toolchain, distribution) cover the production- systems-software claim. Work that's foundational but separable from that claim — the self-hosted compiler rewrite, the Silicon-native interpreter, IDE-level native debugging, LSP integration — moves here.

The TypeScript compiler under src/ is the v1.0 production compiler. Through 1.0, the Silicon interpreter use cases (sgl run, sgl test, sgl eval) are covered via the WAT → wasmtime path; v1.1 adds the Silicon-native surfaces that round out the developer experience and the self-hosted compiler that the language always promised.

Status: v1.1 is officially planned for after 1.0 ships. Stories may move back to 1.0 if priorities shift; new v1.1 work can be appended below. No commitment on ordering or scheduling beyond "after 1.0."

Phase 7 — Self-hosted compiler (rewrite), @comptime + advanced stdlib

Originally: port the complete src/ TypeScript Strata 2.0 surface to a self-hosted boot/ Silicon compiler, add user-level @comptime, and migrate the compiler's internal data structures to Vec/HashMap/Option. Story IDs 3-1 through 3-26 carry the port; CT-1 through CT-5 carry @comptime; story 5a-7 carries the data-structure migration.

Post-2026-05-27: the original boot/ tree was removed and the self-hosted compiler is slated for a future rewrite from scratch. When the rewrite begins, these stories will need re-scoping against the new self-hosted layout (different files, different staging, likely a different compiler architecture) — they're preserved here as the spec the rewrite must satisfy, not as a literal porting checklist.

The src/ TypeScript compiler — which the entire 1.0 toolchain uses (sgl run, sgl build, the CaaS API, the QBE native backend) — already has every surface these stories were going to port. v1.0 ships complete without Phase 7; v1.1 closes the self-hosted loop.

3-26 Port the Phase 5 grammar revisions to the self-hosted parser L, ~8h
Depends on: src/-side revisions landed.

Mirror the four grammar changes shipped in src/grammar/silicon-official.ohm over to the self-hosted parser so the self-hosted compiler accepts the same surface forms the src/ TS compiler does. The revisions to port:

  1. Discard _ identifier in name position (@fn _:Int := 42;, _:T typed-discard parameter slots).
  2. $fn sigil function-type production (:$fn _:R _:T1, _:T2) — surface annotation for Function-kind SiliconTypes, mirroring function-definition shape.
  3. Multi-arg typeArgs whitespace fix — Result[T, E] (with space) must capture both type args.
  4. Parens as optional grouping for function-definition param lists and $fn type annotations.

Also retire the legacy @stratum_operator / @stratum_keyword Elaboration productions, after every built-in stratum has been migrated to the unified @stratum := { ... } form. A byte-equal self-host gate is the binding constraint. Implement as several small commits, each preserving byte-equal, rather than one big bang.

5c-4 Migrate the self-hosted compiler's (ptr, len) pairs to Slice[u8] L, ~6h
Depends on: 5c-3 (Slice[u8] substrate, shipped); 3-26 (self-hosted grammar revisions open for modification)

The src/-side substrate (5c-1/2/3) is complete and runtime-verified; this is the application pass.

7b — @comptime (user-level)

CT-1 Expose strata body evaluator at user-level @comptime M, ~4h
Depends on: Phase 7 AST interpreter

Parser recognises @comptime { ... } as a new expression form. Routes through br_interpret_body with the same scope + Compiler::* dispatch as a stratum handler body.

CT-2 @comptime in expression position (returns a value) M, ~3h
Depends on: CT-1

@let x:Int := @comptime { 6 * 7 };. The block's last expression becomes the inferred value at the call site. Lowering replaces the @comptime expression with the evaluated literal so runtime code carries no trace.

CT-3 @comptime block — push_definition + module mutation M, ~4h
Depends on: CT-2

Top-level @comptime { &Compiler::module::push_definition ... }; works the same way an on::module_finalize handler does. Drains R_PENDING_DEFS into the output module after the main lowering pass.

CT-4 Migrate emitter opcode table to @comptime-generated M, ~4h
Depends on: CT-3, production comptime engine (3-22)

Move the WAT emitter's opcode table from hand-written data to a @comptime block that generates it. This is the litmus test that proves @comptime is strictly more powerful than Zig's — the block calls api.module.push_definition to emit a global. Requires @loop in handler bodies (the table has 100+ entries; unrolling by hand defeats the demonstration).

CT-5 @assert in @comptime blocks S, ~2h
Depends on: CT-2

@comptime { @assert size_of Point == 8; };. If the assertion fails at compile time, emit a diag::error. Useful for invariant checks against struct layouts and constant expressions.

7c — Advanced stdlib (self-hosted migration)

5a-7 Rewrite the self-hosted compiler's internal data structures using Vec/HashMap/Option L, ~8h
Depends on: 5a-6 (HashMap shipped); the self-hosted compiler being open for revision.

The "compiler's internal data structures" named in the roadmap (registry.operators, handler maps, string arenas) are TypeScript objects in src/; the meaningful rewrite is in the self-hosted compiler, which uses hand-coded record types that the new Vec[i32] + HashMap[i32 → i32] stdlib types replace. Phase 11 — Silicon interpreter

Tree-walking interpreter for Silicon, written in Silicon. Drives sgl run, sgl test, sgl eval. Generalises the strata body evaluator from a subset to the full Silicon surface.

Sequencing note: intentionally placed after Phase 7 (self-hosted compiler) and Phase 9 (QBE native backend, shipped in v1.0) so the interpreter can be written in Silicon and compiled to native for production performance. Writing it before native compilation exists would require the interpreter to run interpreted — a 10–100× slowdown that makes it unusable as a dev tool.

Phase 14 — DWARF + LSP

Why this is v1.1, not v1.0: Phase 14 has two halves — 9a (DWARF) for native-binary debugging via gdb/lldb, and 9b (LSP) for IDE integration. Both were marked post-1.0 acceptable in earlier planning: 9a explicitly so (the Silicon interpreter from Phase 11 + sgl eval is the primary debug story); 9b implicitly so (it depends on CaaS Consumer APIs that are partially post-1.0, and most languages ship 1.0 without an LSP — Rust did, Zig did).

Function-name symbols in native binaries come essentially free from QBE → as → ld at 1.0; full DWARF source-line info, variable inspection, and IDE-level support land here when there's concrete demand.

9a — DWARF debug info

The Silicon interpreter (Phase 11) is the primary debug story for v1.0 — it operates on Silicon semantics, shows values as Silicon types, and covers app-developer needs. DWARF wins systems-level debugging of native binaries (crash analysis, profilers, gdb on sigilc itself) — a real but secondary concern.

Function-name symbols in native binaries come essentially free from QBE → as → ld as long as functions are emitted with proper names — so stack traces stay readable without any DWARF work. Full source-line info, variable inspection, and IDE-level native debugging are the v1.1 deliverables.
