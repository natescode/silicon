Sigil 1.1 — User Stories

Stretch work consciously deferred out of 1.0 scope. Phase numbers are retained from v1-user-stories.html so cross-references stay valid; nothing here gates the 1.0 release.

Scope philosophy: 1.0 ships when the four pillars (language, compiler, toolchain, distribution) cover the production- systems-software claim. Tooling that's nice-to-have but not load-bearing for that claim — IDE-level native debugging, LSP integration — moves here. The 1.0 toolchain still includes the Silicon interpreter (sgl eval) as the primary debug story; v1.1 adds the systems-level surfaces that round out the developer experience.

Status: v1.1 is officially planned for after 1.0 ships. Stories may move back to 1.0 if priorities shift; new v1.1 work can be appended below. No commitment on ordering or scheduling beyond "after 1.0."

Phase 9 — DWARF + LSP

Why this is v1.1, not v1.0: Phase 9 has two halves — 9a (DWARF) for native-binary debugging via gdb/lldb, and 9b (LSP) for IDE integration. Both were marked post-1.0 acceptable in earlier planning: 9a explicitly so (the Silicon interpreter from Phase 7 + sgl eval covers Silicon-level debugging, which is the 1.0 story); 9b implicitly so (it depends on CaaS Consumer APIs that are partially post-1.0 themselves, and most languages ship 1.0 without an LSP — Rust did, Zig did).

Moving the whole phase to v1.1 reflects what was already the realistic schedule. Function-name symbols in native binaries come essentially free from QBE → as → ld at 1.0; full DWARF source-line info, variable inspection, and IDE-level support land here when there's concrete demand.

9a — DWARF debug info

The Silicon interpreter (Phase 7) is the primary debug story for 1.0 — it operates on Silicon semantics, shows values as Silicon types, and covers app-developer needs. DWARF wins systems-level debugging of native binaries (crash analysis, profilers, gdb on sigilc itself) — a real but secondary concern.

Function-name symbols in native binaries come essentially free from QBE → as → ld as long as functions are emitted with proper names — so stack traces stay readable without any DWARF work. Full source-line info, variable inspection, and IDE-level native debugging are the v1.1 deliverables.
