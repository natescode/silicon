Review the added defkinds added: platformdecl, exportdecl, localdef etc. They simply call an IR function. I want strata to have more robust hooks into
the compiler. I.E for the def_local. It eventually is implemented with lowerLocalDef. This should be defined in pure Silicon in the Stratum. 
Stratum should be involved with how code is lowered. 
Basically the code can be registered then when the lower.ts runs it looks for the 'def_local' implementation and runs that. The stratum will need
the ability to call the compiler-defined siliconTypeNameToWasm, ctx.pendingLocals.push and ctx.locals.set as well. 
The goal is to have AS LITTLE AS POSSIBLE compiler code and AS MUCH AS POSSIBLE code in Pure Silicon defined in Strata.

As far as I can think of, Sigil needs to do the following only:
  parse all of Silicon
  inherently understand stratum syntax and semantics
  expose IR intrinsics for codegen
  expose numerous compiler library functions and APIs
