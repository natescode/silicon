sigil.wasm

The path from today's stage1.wasm — a self-host test artifact that can compile its own source byte-equal — to sigil.wasm: a single standalone compiler binary that runs under any WASI host and compiles arbitrary Silicon programs end-to-end, with no Bun, Node, or wat2wasm in the loop.

end state one binary, one host, zero bun six phases gates the byte-equal self-host on every step

1. The Goal

A single sigil.wasm file. You install wasmer (or wasmtime, or any WASI Preview 1 host), drop in sigil.wasm, and: 

$ wasmer run sigil.wasm -- delfina.si --target=wasix --out=delfina.wat
  # reads delfina.si from disk via WASI path_open
  # resolves @use chain via the same WASI surface
  # parses, elaborates, typechecks, lowers, emits
  # writes delfina.wat to disk via path_open + fd_write

$ wasmer run delfina.wat
Delfina

No bun, no node, no wat2wasm, no npm install. One compiler binary, one runtime host, one source file in, one WAT file out. The same artifact also self-compiles — running sigil.wasm on the Silicon source of sigil.wasm itself produces byte-equal output.

2. Where We Are Today

Working

  • stage1.wasm (38,262 bytes) exists and runs under wasmer.
  • Self-host fixed point: Stage 1 == Stage 2 byte-equal on its own source.
  • Reads source from stdin, writes WAT to stdout.
  • Compiles the bootstrap test corpus (boot/tests/*.si).
  • The TS Stage 0 compiler (bun run sgl / scripts/build-boot.ts) produces correct output for arbitrary user programs.

Not yet working — what blocks delfina.si-class user programs

Observed by attempting cat strata/*.si modules/wasi_snapshot_preview1.si boot/std/io.si delfina.si | wasmer run stage1.wasm > delfina.wat && wasmer run delfina.wat:

  1. Namespaced extern imports get flattened. Stage1 emits (import "env" "fd_write" ...) for every @extern regardless of whether it lives in src/strata/modules/wasi_snapshot_preview1.si or elsewhere. But call sites still mangle to $wasi_snapshot_preview1_fd_write, producing an unresolvable reference at link time.
  2. No _start synthesis. Top-level expressions don't get wrapped in a WASI entry-point function. Even if the imports were correct, wasmer would have no _start to call.
  3. No CLI surface. Stage1 reads stdin and writes stdout — there is no argv parsing, no file open, no --out= handling, no @use resolution. The host (currently scripts/run-silicon.ts in Bun) does that work.
  4. Strata bundle is supplied externally. Stage1's behaviour depends on the caller concatenating src/strata/*.si in front of the user source. The bootstrap doesn't know its own strata; they're injected at runtime by the host.

Each of these is small in isolation. Together they're the difference between "a self-host test that round-trips" and "a compiler you can use".

4. Architecture — What sigil.wasm Looks Like Inside

The same compiler pipeline you already have, with the host-side responsibilities folded into the wasm: 

┌────────────────────────────────────────────────────────────────────────┐
│  sigil.wasm  (single artifact, ~80–120 KB target)                    │
│                                                                        │
│  ┌─────────────────────────┐                                           │
│  │  Embedded source bundle │  <-- baked in at sigil.wasm build time    │
│  │  - all strata/*.si      │      via a build script (see §6); the     │
│  │  - all modules/*.si     │      bundle lives as a memory-resident    │
│  │  - boot/std/*.si        │      length-prefixed UTF-8 blob.          │
│  └─────────────────────────┘                                           │
│                ▼                                                       │
│  ┌────────────┴────────────────────────────────────────────────────┐   │
│  │  CLI & I/O layer (Silicon source)                               │   │
│  │  - args_get + args_sizes_get  ──▶  parse flags / paths          │   │
│  │  - path_open + fd_read         ──▶  read user source            │   │
│  │  - @use resolver (in Silicon)  ──▶  recurse on imports          │   │
│  └────────────────┬────────────────────────────────────────────────┘   │
│                   ▼                                                    │
│  ┌────────────────┴────────────────────────────────────────────────┐   │
│  │  Compiler pipeline (existing boot/* — same code that self-hosts) │   │
│  │  lex ──▶ parse ──▶ load_strata ──▶ elaborate ──▶ lower ──▶ emit │   │
│  └────────────────┬────────────────────────────────────────────────┘   │
│                   ▼                                                    │
│  ┌────────────────┴────────────────────────────────────────────────┐   │
│  │  Output                                                         │   │
│  │  - WAT text written to --out= path via path_open + fd_write     │   │
│  │  - diagnostics to fd 2 (stderr)                                 │   │
│  │  - exit code via proc_exit                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────────────────────────┘

The two genuinely new pieces are the embedded source bundle (which makes the binary self-contained) and the CLI & I/O layer (which replaces the Bun-side host script). Everything else is code that already exists in boot/ and is exercised by today's self-host.

5. Phased Rollout

Six phases. Each is independently shippable — if any phase lands and the next is delayed, the project is in a better, not worse, state.

Phase 1 Close the WAT-correctness gaps — stage1 can compile WASI user programs
Effort
~3–5 days
Risk
Medium — touches the emitter, which is the byte-equal-critical path.
Gate
All existing self-host tests still byte-equal. cat <strata> <modules> boot/std/io.si delfina.si | wasmer run stage1.wasm | wasmer run - prints "Delfina".
  1. Teach boot/elab/elaborator.si to record the module string for each @extern declaration — today everything inherits the default "env". Use the namespace prefix in module::name calls to drive the lookup.
  2. Update boot/emit/wat.si to emit (import "<module>" "<name>" ...) using the recorded module and to keep the mangled $<module>_<name> in sync between import and call site.
  3. Add a --target=wasix flag awareness to boot/ir/lower.si (read from a global until argv lands in phase 4). When set, wrap top-level non-definition statements into a synthesised _start function exported as "_start".
  4. Audit @export handling in boot/emit/wat.si; ensure exported functions and globals reach the WAT export list.
  5. Add a fixture: boot/tests/delfina_smoke.si — the minimal "print a string" program. Compiled by stage1, then run by wasmer, must print the expected line.
Phase 2 Embedded source bundle — strata + modules baked in
Effort
~2–3 days
Risk
Low — new code, doesn't touch the emitter.
Gate
No host-side strata concatenation needed. wasmer run sigil.wasm < delfina.si works without prepending strata files.
  1. New build script (one of: scripts/build-sigil.ts for Stage 0 bootstrap; later replaced by a Silicon-side equivalent): reads every src/strata/*.si and src/strata/modules/*.si, concatenates with file headers, emits a length-prefixed UTF-8 blob as a Silicon @let bundle :String := '...' in a generated boot/embedded_bundle.si.
  2. Adjust boot/stage1.si (renamed boot/sigil.si at this point) to consume the embedded bundle before reading user source from stdin: parse + register strata from the bundle, then proceed to user input.
  3. The bundle is purely additive — if the user wants to override / extend strata, they can still concatenate their own in front. But for typical use, nothing extra is required.
  4. Rename the build artifact: stage1.wasm → sigil.wasm. Keep stage1.wasm as a build-product alias during migration.
Phase 3 Bootstrap-side @use — via the stratum machine
Effort
~3–5 days — companion to use-as-stratum-plan.html Phase B
Risk
Medium — touches the parser-side AST + a new resolver pass.
Gate
User programs with @use 'path.si'; compile without manual concatenation. Cycle + missing-file errors raised with source locations.
  1. Implement the @stratum_keyword Use stratum on the Stage 0 side first (per use-as-stratum-plan.html) so the design is proven.
  2. Port the same logic into boot/elab/use.si: the bootstrap-side stratum body uses &Compiler::import, which the bootstrap implements in Silicon (~30 LOC) using path_open + fd_read to load the referenced file, then recurses into the parse / register loop.

Phase 4 CLI & file I/O — sigil.wasm takes argv
Effort
~2–3 days
Risk
Low — new layer on top of working compiler.
Gate
wasmer run sigil.wasm -- delfina.si --target=wasix --out=delfina.wat works end-to-end without stdin/stdout indirection.
  1. Wire args_sizes_get + args_get in boot/std/argv.si (new). Return a vec of argument strings.
  2. Implement path_open + fd_read based file reads in boot/std/io.si — today io.si only knows about stdin/stdout. The u64 rights-flags issue from the bootstrap plan (Phase 0 note in boot/main.si) needs the i64 surface from docs/bootstrap-plan.html or a workaround using two i32s if it isn't ready.

Phase 5 Typechecker in the bootstrap
Effort
~1–2 weeks — the biggest single phase
Risk
High — type semantics are the most subtle part of the compiler. The byte-equal gate gets stricter, not looser, because typecheck failures change the lowered output.
Gate
Compile-time error parity with Stage 0 on the existing test corpus. Adding a type error to any boot/tests/*.si file is caught by sigil.wasm with the same diagnostic Stage 0 raised.
  1. Today's bootstrap is a pure lowering pipeline — it trusts that the input typechecks. That's fine for self-host (the bootstrap source is known-good) but means sigil.wasm doesn't catch user type errors. Phase 5 ports src/types/typechecker.ts to Silicon: AST walk, symbol table, SiliconType representation, the diagnostic factories from src/types/errors.ts.
  2. Reuse the same error codes (E0001…E0007) so tooling around diagnostics (LSP, tests) doesn't have to fork.
  3. This phase can land after phases 1–4: the resulting compiler will compile correct programs identically and trap on malformed ones via runtime traps rather than typecheck errors. Phase 5 upgrades that to clean compile-time diagnostics.
  4. The lift here is roughly the size of all of phases 1–4 combined; it deserves its own design pre-read.
Phase 6 End-to-end fixed point + Wasmer-registry publish
Effort
~2–3 days
Risk
Low — integration / packaging.
Gate
sigil.wasm compiles its own source, output byte-equals the previous sigil.wasm, and a third-party user who installs natescode/sigil from the Wasmer registry can compile delfina.si with zero source-code dependency on this repo.
  1. End-to-end self-host: wasmer run sigil.wasm -- boot/sigil.si --target=wasix --out=sigil.wat → the sigil.wat assembles to a sigil.wasm that is byte-equal to the input artifact. This is the new "Stage 1 == Stage 2" gate but at the full user-program-compiling level, not just the self-compile level.
  2. CI: add a gate that runs the round-trip on every commit. Any drift in the emitter shows up immediately as a non-zero diff of the two sigil.wasms.
  3. Publish to the Wasmer registry as natescode/sigil — one wasmer install natescode/sigil then wasmer run natescode/sigil delfina.si. This is the externally-visible deliverable that says "100% bootstrapped".
  4. Retire the TS Stage 0 compiler as production path. Keep it for development tooling (LSP, fast iteration during compiler-internal changes), but the canonical compiler users invoke is sigil.wasm.

6. The Embedded-Strata Question — Build-Time vs. Runtime

Two viable shapes for how built-in strata reach the compiler. The plan above assumes embedded, but the alternative deserves a quick mention because it changes the install story.

Embedded (recommended)

Strata source baked into sigil.wasm at build time as a length-prefixed blob in the data section. Loaded into memory at _start, parsed, registered.

Pros: Single-file install. Reproducible — sigil.wasm has its own strata, no version drift. No "where do I put the strata folder?" question.
Cons: Slightly larger binary (~10–20 KB of compressed source). Updating strata means rebuilding sigil.wasm.

Disk-loaded

sigil.wasm reads strata/ from a directory at runtime ($SIGIL_STRATA env var, or convention path like ./strata/).

Pros: Hack on strata without rebuilding the compiler. Smaller binary. Matches the Stage 0 dev loop.
Cons: Two-piece install. Implicit dependency on filesystem layout. Failure modes if the strata path is wrong or partial.

Recommendation: embed by default; expose a --strata-dir=<path> flag that overrides the embedded bundle when developers need to iterate on strata without rebuilding. Best of both.

7. The Load-Bearing Invariant — Self-Host Byte-Equality

Today's stage1.wasm → stage1.wat → stage1.wasm round-trip produces byte-equal output (38,262 bytes). That invariant is the sign that the compiler is faithfully implementing its own semantics. Every phase above must preserve it.

Phases 1, 3, and 5 touch code that affects emitted WAT. Each needs a CI step that rebuilds sigil.wasm twice and diffs the bytes. A non-zero diff fails the phase. Phase 6 extends this invariant to the user-program compile path: sigil.wasm compiling its own source must produce byte-equal output, not just on the self-host but on a separate round-trip after every change.

8. What This Plan Does Not Cover

  • Performance. The bootstrap is presumed "fast enough" today (the self-host compiles in well under a second). Optimisation can come post-Phase-6.
  • i64 / u64 support. The WASI path_open rights-flags are u64; today's Silicon i32-only world side-steps them. The bootstrap plan (docs/bootstrap-plan.html) has the i64 work as a separate Phase. Phase 4 here may need to gate on it — or apply the documented workaround (extern signature using two i32 halves).
  • Module system semantics beyond @use. Selective imports, namespacing, package-manager indirection — tracked in use-as-stratum-plan.html non-goals, intentionally deferred.
  • Dynamic imports / loading at runtime. Separate design slice; the static module signatures built here become the type of dynamic handles later.
  • Multiple target backends. Today's emitter is WAT-only. A second backend (binaryen-style binary, JS, native) is post-bootstrap.
  • Standard library beyond boot/std/*.si. What lives in the stdlib is its own design conversation — this plan only commits to the surface needed for the compiler itself plus the I/O primitives the CLI uses.

9. What Lands First

Phase 1. It is the smallest slice that meaningfully changes what stage1.wasm can do — from "compiles its own source" to "compiles any WASI user program". Three concrete code changes (extern module tracking, _start synthesis, @export wiring) plus one fixture file (delfina_smoke.si). The byte-equal gate stays at today's 38,262 bytes for the self-host round-trip; the new fixture proves the new behaviour.

Phase 1 is a satisfying milestone on its own: the moment it lands, the answer to "can I use the bootstrap to compile delfina.si?" changes from "not quite" to "yes — cat strata/*.si modules/*.si boot/std/io.si delfina.si | wasmer run stage1.wasm | wasmer run -". Phase 2 then makes that one-liner shorter; Phase 4 makes it a real CLI invocation.
