Silicon-Only Bootstrap

Path from today's TypeScript-driven Stage 0 to a repo where every Sigil development task — adding language features, running tests, producing wasm artifacts — happens in Silicon alone. The deletion of src/ is the last step, not the first.

What this doc is. A staged plan with concrete gates so each phase lands as its own merge-able PR while the TypeScript backstop still works. No phase requires the previous phase to be deleted; the cutover (Phase 7) only fires after every replacement is proven by a byte-equal gate or executable test.

Definition of done

  • A fresh clone of the repo contains only: boot/*.si, a tiny shell/PowerShell driver, and a checked-in wasm-bin/stage1.wasm seed. No src/, no package.json, no bun.lockb, no node_modules.
  • External tool requirements: wasmtime ≥ 14 on PATH; wat2wasm from wabt on PATH. Nothing else.
  • ./build.sh (or ./build.ps1) rebuilds stage1.wasm from boot/*.si using the prior stage1.wasm as the compiler. The new artifact is byte-equal to the seed (fixed-point check).
  • ./build.sh test runs the full Silicon-side test corpus (a Silicon-driven harness, no bun test).
  • Adding a new @stratum_keyword to a boot/strata/*.si file is sufficient to change stage1's output. No TS or C glue.

Today's audit (2026-05-17)

Critical: stage1.wasm today is a bootstrap fixed point for the boot/* tree itself, not a general Silicon compiler. Several systems still live exclusively in TypeScript and must move to Silicon (or to a tiny shell driver) before TS can be removed.

Strategic principle

Every phase ends with the repo still buildable via the existing TS toolchain and via the new Silicon-only path. The two paths produce byte-equal artifacts as the gate. We never have a "soft" cutover; the deletion in Phase 7 is mechanical once all gates are green.

Phase 1
Body interpreter — full &Compiler::* surface
Scope
Extend boot/elab/body.si to evaluate rich strata bodies: scope, multi-segment Node.x.y paths, @local bindings, and dispatch of every &Compiler::* call to a Silicon-backed implementation.
Touches
boot/elab/body.si (+~300), boot/compiler_api/*.si (new, ~600 across 17 ops), boot/ir/lower.si (refactor: extract scope/locals/globals as Compiler-API providers).
Gate
Stage 1 emits byte-equal WAT vs Stage 0 for the existing src/e2e/examples/*.si corpus, with boot/ir/lower.si patched to dispatch each rich-body keyword through interpret_body instead of its current hard-coded branch.
Why first
Every later phase depends on it. Until rich bodies actually execute in Silicon, the strata in src/strata/*.si are dead text inside stage1 and boot/ir/lower.si is the real source of truth.

Subslices (each landable on its own branch)

  1. 1a — Scope & multi-segment path eval. Add a scope vec to body.si, support @local x := expr; bindings, evaluate Node.foo.bar by walking AST_* field offsets. Tested in isolation; not yet wired into lower.si.
  2. 1b — Compiler::ir::* constructors. 8 entry points that just call the existing ir_* builders in boot/ir/nodes.si: makeFunction, makeGlobal, makeImport, makeLocal, makeExport, makeIf, makeLoop, makeConst, makeBreak, makeContinue, makeReturn.
  3. 1c — Compiler::ctx::* state surface. globals::set, varNames::add, pendingLocals::push, locals::set, loopStack::{push,pop,peek}, nextLoopId. These are thin wrappers over the existing per-file globals in boot/ir/lower.si; the refactor moves them into a shared boot/compiler_api/ctx.si.
  4. 1d — Lowering helpers. lowerExpr, lowerExprIfDefined, arg, watId, resolveType, resolveFunctionReturnType, lowerParams, lowerFunctionBody, lowerGlobalInit, lowerExternParams, lowerExternResult, assertDefined, choose, isVarName, expandMatchChain.
  5. 1e — Wire-up + byte-equal gate. Replace the hard-coded lower_definition branches in boot/ir/lower.si with calls to interpret_body on the keyword's elaborated semantics block. Diff stage1 WAT vs stage0 WAT byte-for-byte across the e2e corpus.
Gate 1   For every src/e2e/examples/*.si fixture: stage1.wasm < fixture.si produces WAT byte-equal to compileToWat(typecheck(elaborate(parse(fixture)))). Plus stage1==stage2 self-host stays green.
Phase 2
Typechecker port
Scope
Port src/types/typechecker.ts (1100 LoC of TS) to boot/types/*.si. Inference for literals, binops, calls, locals, blocks, returns; intrinsic signatures via the strata registry.
Touches
boot/types/{types,checker,errors,intrinsicSig}.si (new); minor hook in boot/stage1.si between parse and lower.
Gate
For every fixture in src/types/typechecker.test.ts + property-test corpus, stage1's type-error JSON dump matches Stage 0's, byte-for-byte.

Sequencing risk. The typechecker is the single largest port. Split it into slices mirroring TS file structure: types.si (the SiliconType ADT), intrinsicSig.si, errors.si, checker.si (top-down walker). Each slice ships with its own byte-equal gate against the corresponding TS module's exposed behaviour.

Phase 3
@use resolver + multi-file entry
Scope
Port src/modules/useResolver.ts (cycle-safe, depth-first @use walk) to boot/modules/use.si. Update stage1.si's entry so that, given a single source-file path, it resolves the @use graph from disk via path_open + fd_read rather than expecting a concat bundle on stdin.
Gate
wasmtime --dir . stage1.wasm boot/main.si compiles the same WAT as the existing host-concat path.
Phase 4
WAT-to-WASM step
Decision
Shell out to wat2wasm (from wabt). Pure encoder in Silicon is ~800 LoC of fiddly leb128 + section ordering — deferred to a Phase 4-bis if external wabt becomes unacceptable.
Touches
build.sh, build.ps1 — shell drivers that run wasmtime stage1.wasm < bundle | wat2wasm - -o out.wasm.
Gate
Same byte sequence as watToWasm() in src/codegen/toWasm.ts on every fixture; stage1.wasm produced by the shell path equals the one produced by scripts/build-stage1.ts.

Why wabt is acceptable. wat2wasm is a single static binary, redistributable, and already required for the existing test suite. Treating it as a "platform" tool (like wasmtime itself) keeps the Silicon-only repo honest without growing a second 800-LoC subsystem.

Phase 5
Shell-driven build & self-host loop
Scope
Replace scripts/build-stage1.ts + scripts/run-silicon.ts with a single shell driver that uses only the prior stage1.wasm as the compiler.
Touches
build.sh, build.ps1, optional Makefile.
Gate
Fixed-point: ./build.sh stage1 produces a wasm byte-equal to the seed stage1 in wasm-bin/; ./build.sh stage1 stage2 produces wasm byte-equal to that.
# build.sh (sketch)
#!/usr/bin/env bash
set -euo pipefail
STAGE_IN=${1:-wasm-bin/stage1.wasm}
STAGE_OUT=${2:-wasm-bin/stage1.next.wasm}

# Concat boot/ tree in dependency order. boot/build_order.txt is checked in.
cat $(cat boot/build_order.txt) > /tmp/sigil-bundle.si

# Compile with the existing stage1 → WAT.
wasmtime --dir . "$STAGE_IN" < /tmp/sigil-bundle.si > /tmp/stage.wat

# Assemble.
wat2wasm /tmp/stage.wat -o "$STAGE_OUT"

# Self-host check.
cmp "$STAGE_IN" "$STAGE_OUT" && echo "fixed point ✓"
Phase 6
Silicon-side test runner
Scope
boot/test_runner.si: walks a tests dir, runs each *.test.si's entry under stage1, asserts that the printed output matches a sibling *.expected file. Port the most critical TS tests as Silicon fixtures.
Gate
All Phase 1/2/3 gate tests now run under the Silicon runner. ./build.sh test exits 0 on a clean tree.
Phase 7
Delete TypeScript
Scope
Mechanical removal. Delete src/, scripts/*.ts, tests/ (after porting what's needed), package.json, bun.lockb, tsconfig.json, node_modules. Update README + CLAUDE.md.
Pre-flight
Phases 1–6 all green on CI. Seed stage1.wasm committed to wasm-bin/ (yes, binary in git — same precedent as boot.wasm today).
Gate
Fresh clone → ./build.sh → byte-equal stage1; ./build.sh test → exit 0. Nothing else.

Risks & mitigations

RiskMitigation
Phase 1 introduces subtle ordering bugs that break stage1==stage2.Each subslice (1a–1e) ships behind a feature flag in boot/ir/lower.si — old hard-coded path stays as fallback until 1e's byte-equal gate passes on the full corpus.
Typechecker port (Phase 2) drifts from Stage 0's error messages.Compare JSON-mode output, not human strings. Stage 0 already has --diagnostics=json; stage1's port emits the same schema.
Bootstrapping from zero (no prior stage1.wasm).Seed wasm-bin/stage1.wasm stays in git. Treat it like a Rust compiler treats its seed binary. Updating it requires a ./build.sh stage1 stage2 && cp … incantation, intentionally a two-step ritual.
Grammar gap discovered late.Run the Stage 0 corpus through stage1's parser on every PR; failing fixtures gate the merge. Already wired up (38/40 byte-equal today).
Loss of property tests / fuzz coverage.Out of scope for Phase 7. The Silicon-side runner doesn't replace property tests on day one; treat fuzz/property suites as a separate (TS-bound) CI job that can stay in a sibling tooling repo if needed.

What lands on the bootstrap/silicon-only branch today

  • Stub boot/compiler_api/ directory with a README mapping each of the 17 entry points to its TS source location, so subsequent slices have a clear target.
