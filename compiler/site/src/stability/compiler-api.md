# CompilerAPI stability

The TypeScript-side `CaaS` surface — what library consumers import as
`sigil/caas` — is stable at 1.0. Internal subsystems are not.

| Surface | Stability | Versioning |
|---------|-----------|------------|
| `src/caas/index.ts` `@public` exports | Stable | Additive in minor, breaking in major |
| `src/api.ts` re-exports | Stable (canonical import path) | Tracks `src/caas/` |
| `etc/sigil.d.ts` rolled-up types | Stable | Tracks `src/caas/` |
| `_*`-prefixed exports | `@internal` | Free to change |
| `src/elaborator/`, `src/types/`, `src/codegen/` direct imports | Internal | Free to change; do not import |

## Add / change / remove rules

| Change | Allowed in | Requires |
|--------|-----------|----------|
| Add a new `@public` export | Any minor release | Changeset entry |
| Add an optional field to a public type | Any minor release | Changeset entry |
| Add a required field to a public input type | Major release | Migration note |
| Remove or rename a `@public` export | Major release | Deprecation in prior major; migration note |
| Change a `@public` function's return type | Major release | Migration note |
| Refactor anything inside `src/elaborator/` etc | Any patch | None |

See [`docs/stability.md`](/stability/) §2 for the full rule table and
[`docs/api-boundaries.md`](/reference/api-boundaries) for the per-subsystem
import rules.

## Rolled-up surface

`etc/sigil.api.md` (committed) is the single source of truth for the
type surface, generated by
[api-extractor](https://api-extractor.com/). Any drift requires
regenerating the report (`npm run api:extract`); the diff is
reviewer-visible.
