**1.0 surface lockdown begins.** Three sources of truth landed:

- `etc/sigil.d.ts` + `etc/sigil.api.md` — the rolled-up TypeScript surface,
  generated by api-extractor and committed. Any drift in the public CaaS API
  now requires regenerating the report (`npm run api:extract`) and a
  reviewer-visible diff.
- `wit/comptime.wit` — the comptime host-import contract that
  `src/comptime/imports.ts` and downstream Silicon `@extern` declarations
  must conform to. Closes the "three places to keep in sync" problem
  (TS host, Silicon externs, API docs).
- `docs/adr/` — Architectural Decision Records for the seven open
  pre-1.0 architectural questions (`@generic` scope, legacyBlockTranslator,
  comptime engine path, Backend interface, JS collections in surface,
  WASM direct emitter, diagnostic renderer). Each ADR is ~1 page with
  options + recommendation; resolution comes in subsequent PRs.

No runtime behavior changes.
