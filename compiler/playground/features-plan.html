Feature 1

Auto-run Main Checkbox

A checkbox in the header. When checked, the playground automatically calls the exported main function (zero-argument, any return type) immediately after a successful compile — no need to open the exports panel and click Call.

UI

  • ☐ Checkbox placed in the header, right of the theme dropdown, left of the status dot. Label: run main
  • • State persisted to localStorage so it survives page refresh.

Logic

  1. 1
    After compile succeeds — check if checkbox is checked.
  2. 2
    Find main in exports — look for an export named main with zero parameters. If found, proceed.
  3. 3
    Call it — invoke wasmInstance.exports.main(), flush the char buffer, and log the return value (if non-void) to the console.
  4. 4
    Game loop convention — if tick is also exported (zero-arg), start a setInterval at 100 ms. If handle_key is exported (one Int param), wire document.keydown to it. Both stop when the next compile begins.
Export conventions detected at compile time
  • main() — called once on auto-run
  • tick() — called every 100 ms (game loop)
  • handle_key(key: Int) — wired to keydown event, receives e.keyCode

Feature 2

Collapsible HTML Render Panel

A right-side panel containing a <canvas> element (for graphics) and a <div id="html-out"> container (for DOM output). Silicon programs drive it through new web:: API functions. The panel is hidden by default and opens via a toggle button.

Panel layout

  • Fixed width: 360 px when open; collapses to a 28 px toggle rail.
  • Toggle button (◁ / ▷) lives on the rail, vertically centered.
  • Panel header shows "Render" label and a Clear button.
  • A <canvas id="render-canvas" width="320" height="320"> sits at the top.
  • A <div id="html-out"> sits below for HTML string injection.
  • Canvas size configurable in the future; 320×320 is the default.

New web:: APIs

These are added to src/strata/modules/web.si and implemented in playground/web-env.js.

Canvas drawing

canvas_set_fill r:Int, g:Int, b:Int— set fill color (RGB 0-255)
canvas_set_stroke r:Int, g:Int, b:Int— set stroke color
canvas_fill_rect x:Float, y:Float, w:Float, h:Float— filled rectangle
canvas_stroke_rect x:Float, y:Float, w:Float, h:Float— stroked rectangle
canvas_clear_rect x:Float, y:Float, w:Float, h:Float— clear region to transparent
canvas_fill_text ptr:String, x:Float, y:Float— draw text string
canvas_set_font ptr:String— set font (e.g. '16px monospace')
canvas_width:Int— query canvas pixel width
canvas_height:Int— query canvas pixel height

HTML output

set_html ptr:String— set innerHTML of #html-out
clear_html— clear #html-out

Playground JS behavior

  • When any canvas or HTML API is used, the render panel opens automatically.
  • Clicking the Clear button calls ctx.clearRect and clears #html-out.
  • The panel background is white for the light theme, #0d1117 for dark themes.

Feature 3

Three New Examples

A — Simple Main

Demonstrates the @fn main convention: a zero-argument entry point that calls helper functions and logs results to the console. No new APIs needed.

simple_main
core
# Entry point example — call helpers from main and print results.

@fn add a:Int, b:Int := { a + b };

@fn factorial n:Int := {
    &@if n <= 1, { &@return 1 };
    n * (&factorial (n - 1))
};

@fn main := {
    @local sum     := (&add 10, 32);
    @local fac10   := (&factorial 10);
    &web::console_log sum;
    &web::console_log fac10
};

@export main;

B — HTML TODO List

Uses web::set_html to render a styled list into the render panel. Demonstrates string building and the HTML output API. The program builds a static list at startup in main.

C — Snake

A classic Snake game on the canvas. Uses the game-loop convention: main initialises state, tick runs each frame, and handle_key receives arrow key codes. The JavaScript runtime detects all three exports and wires them up automatically when run main is checked.

How the JS runtime drives the game
  • After compile, if main + tick + handle_key are detected → open render panel
  • Call main() once (sets up initial state and draws first frame)
  • Start setInterval(tick, 120) — 120 ms ≈ 8 fps, good for Snake
  • Wire document.addEventListener('keydown', e => handle_key(e.keyCode))
  • Both interval + listener are torn down on the next compile
snake
canvasweb api
# Snake — uses the tick + handle_key export convention.
# Enable "run main" and compile.

@let GRID   := 20;    # cells per side
@let CELL   := 16;    # px per cell

@var head_x := 10;
@var head_y := 10;
@var dir    := 0;    # 0=right 1=down 2=left 3=up
@var food_x := 5;
@var food_y := 15;
@var alive  := 1;
@var length := 3;

@fn cell x:Int, y:Int, r:Int, g:Int, b:Int := {
    &web::canvas_set_fill r, g, b;
    &web::canvas_fill_rect
        (&@toFloat (x * CELL)),
        (&@toFloat (y * CELL)),
        15.0, 15.0
};

@fn draw := {
    # Background
    &web::canvas_set_fill 13, 17, 23;
    &web::canvas_fill_rect 0.0, 0.0,
        (&@toFloat (GRID * CELL)),
        (&@toFloat (GRID * CELL));
    # Snake head
    &cell head_x, head_y, 63, 185, 80;
    # Food
    &cell food_x, food_y, 248, 81, 73
};

@fn main := {
    head_x = 10; head_y = 10;
    dir = 0; alive = 1; length = 3;
    &draw
};

@fn tick := {
    &@if alive == 0, { &@return };
    # Move head
    &@if dir == 0, { head_x = head_x + 1 };
    &@if dir == 1, { head_y = head_y + 1 };
    &@if dir == 2, { head_x = head_x - 1 };
    &@if dir == 3, { head_y = head_y - 1 };
    # Wall collision
    &@if (head_x < 0) || (head_x >= GRID), { alive = 0; &@return };
    &@if (head_y < 0) || (head_y >= GRID), { alive = 0; &@return };
    # Food pickup
    &@if (head_x == food_x) && (head_y == food_y), {
        length = length + 1;
        food_x = (food_x + 7) % GRID;
        food_y = (food_y + 13) % GRID
    };
    &draw
};

@fn handle_key key:Int := {
    &@if key == 37, { dir = 2 };  # ←
    &@if key == 38, { dir = 3 };  # ↑
    &@if key == 39, { dir = 0 };  # →
    &@if key == 40, { dir = 1 }   # ↓
};

@export main;
@export tick;
@export handle_key;
