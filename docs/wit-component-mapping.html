WIT ↔ Silicon: components, worlds, interfaces & capabilities

A Silicon component is a WIT package; its single world is the contract — imports = required capabilities, exports = what it provides. Interfaces are named bundles of functions and types. Capabilities cross the boundary as unforgeable resource handles: comptime-checked inside the component, runtime-enforced at the edge.

The mapping

Capabilities, specifically

  • Imports are the authority manifest. world { import wasi:filesystem; } means "this component must be granted filesystem access." No import ⇒ no authority — there is no ambient open(). This is ADR-0015's "you must be handed a capability," lifted to the component level.
  • Mint & hand out = export a resource + a factory func(…) -> own<cap>. The mint-site stays inside (its constructor isn't @pub); only an opaque handle crosses.
  • own vs borrow = ADR-0011 rcaps — a consumed capability lowers to own<T>, a borrowed one to borrow<T>.
  • A capability must be a resource, never a record — a record's fields are readable and reconstructable, so it would be forgeable. WIT has no separate "capability" concept (and no effect rows, matching ADR-0015): resources + the no-ambient-imports model are the mechanism. Reuse them; don't invent a parallel encoding.
  • The boundary is the comptime→runtime handoff. Comptime mint-site checking only reaches within one compilation unit, so across separately-built components unforgeability must become runtime (the handle table). The same FsCap is a free comptime value inside, a tracked resource handle at the edge.

One example, both sides

Silicon (the component)

# sgl.toml
#   [package] namespace="acme"
#             name="textstats" version="0.1.0"
#   [dependencies]  # wasi caps the world imports

# src/main.si — ROOT module = the world
\\ @pub @export count_words (Str) -> Int
@fn count_words text := { word_count(text) };

# a capability VALUE is passed in — never ambient
\\ @pub read_all (FsCap, Str) -> Str
@fn read_all cap, path := { fs_read(cap, path) };

Generated WIT

package acme:textstats@0.1.0;

world textstats {
  import wasi:filesystem/types@0.2.0;   // authority IN
  export count-words: func(text: string) -> u32;
}

// a minted, handed-out capability:
interface sessions {
  resource session;                       // 🔑 unforgeable
  open: func(key: string) -> own<session>;  // factory
  rows: func(s: borrow<session>) -> u32; // rcap
}
v1.0 status. The default build statically merges every module — and, by default, dependency components — into one core .wasm, because neither browsers nor Bun load Component-Model components natively yet. So at v1.0 there is no runtime boundary in the shipped artifact: capabilities are enforced at comptime only (ADR-0013/0015), and statically-merged deps stay in that same comptime-trust regime. The WIT/resource encoding above is the forward-looking --emit=component design (deferred past v1.0); --emit=wit can declare these shapes as forward-compat metadata before anything enforces them at runtime.
