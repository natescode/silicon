Executive Summary

Silicon's defining architectural bet is that syntax is not semantics: operators and keywords are not baked into the grammar but registered as *data* in @stratum definitions under compiler/src/strata/*.si. A + is not a parser production — it is Compiler::register::operator('+') plus an on::lower('+', Plus_lower) handler whose body builds an IRBinOp with i32.add. The hand-written LL(1) parser (compiler/src/parser/handwritten/parser.ts) never grows per-feature; new operators and keywords ride the existing Definition/FunctionCall forms and are given meaning by strata the comptime engine compiles to WASM and runs at lower time. This is the "syntax ≠ semantics" thesis made real, and for the two families where it matters most — operators and control flow — it is genuinely, verifiably data-driven. Every operator in operators.si/bitwise.si/logic.si/strings.si and every control keyword in if.si/loop.si/match.si/control.si carries a real on::lower handler, because the registration stub deliberately sets no intrinsic (strataLoader.ts:298-300), so the !intrinsic branch in lowerBinaryOp (lower.ts:1295) fires the handler rather than a hardcoded path. The migration is far enough along that strataLoader.test.ts:392 asserts registry.expanders.size === 0 for built-ins — the legacy TS expander-map dispatch is now dead code.

The hard numbers tell the honest story. Roughly 45+ operator/cast/logic surfaces are fully data-driven strata: the Int/Float/Int64/UInt arithmetic-and-comparison families (operators.si:9-441), the four bitwise ops (bitwise.si), ||/@and/@or/@not (logic.si), ++ (strings.si), all six casts (cast.si), and the def-kind keywords @global/@fn/@local/@extern/@type/@enum/@type_alias/@type_distinct/@struct/@platform/@export. Against that, only 3 of the 9 spec'd StrataTypes are live and data-driven (Operator, Control, Metadata); a 4th (Type) is realized indirectly through the code-level Definition variant plus hardcoded primitives; and 4 of 9 are entirely unbuilt (Constraint is a dead enum slot, Runtime/Capability/DSL have no enum variant at all). The code enum (strataenum.ts:2-21) has 6 variants {Keyword, Operator, Control, Definition, Constraint, Metadata} that do not line up with the spec's 9-name table in docs/strata.md:16-26 — a doc-vs-code drift that should be reconciled.

Between "pure data" and "pure primitive" sits the dissolution-in-progress: five keywords (@try, @with_arena, @move_to_parent_arena, @fnref, @call_indirect) register their *surface* via register::expression_keyword in control.si:46-81 but have no on::lower handler — their lowering is hardcoded in ir/lower.ts (lowerTry:1670, lowerWithArena:1885, lowerMoveToParentArena:2023, lowerFnRef:1589, lowerCallIndirect:1625). These are not stubs; they are complete, tested implementations. They remain TS because each needs a CompilerAPI capability the handler interface does not yet expose: fresh-local allocation + multi-statement Block synthesis (@try — now available), heap-pointer save/restore + AST tail-walking (@with_arena), and funcref-table/type-section mutation (@fnref/@call_indirect). They are the visible edge of the still-reserved Codegen StrataType.

A clean must-stay-primitive substrate anchors the whole system and is circular-by-necessity: the LL(1) parser and lexer (the fixed grammar strata are parsed *by*), the comptime engine (comptime/engine.ts — the machine that *makes* strata executable), the CompilerAPI bridge (compiler-api/index.ts — the surface strata are written *against*), the WASM::*/IR::* intrinsic alphabet (intrinsics/intrinsics.ts — the leaf ops strata reference *by name*), and the WASM byte emitter + LEB128 (codegen/wasm-emitter.ts — the codegen *floor*). None of these can be a stratum without circularity. The two whole-program memory modes (arenas/no-GC and --target=wasm-gc) and the WASI preview1 surface are built and tested; but the entire capability/borrow/effect stack (ADRs 0011/0012/0013/0015) and the entire FFI chain (ADRs 0017/0018/0019) are Proposed design with zero compiler code — and that FFI chain is the single hard v1.0 product gate.

The Three Buckets (the answer to the audit question)

1. Implemented via Strata — what's genuinely data-driven today

These features live as @stratum data in .si files; their lowering is expressed in on::lower handler @fns (or, for definition kinds, delegated to a *documented* CompilerAPI host-expander). They are the proof the model works.

Operators (fully data-driven). Every binary operator is a stratum building an IRBinOp via compiler::ir_makeBinOp:

  • Int arithmetic + comparison (+ - * / % == != < > <= >=) — operators.si:9-84, Plus_lower → i32.add at line 14.
  • Float overloads via register::typed_operator('<op>','Float') compound keys → f32.* — operators.si:86-160.
  • Int64 overloads → i64.* — operators.si:162-239.
  • Unsigned UInt8/16/32/64 div/mod/shr + comparisons → *_u variants — operators.si:241-441 (DivideU8_lower → i32.div_u at 252; DivideU64 → i64.div_u at 399). By design, + - * == != are *not* overloaded for unsigned (same WAT as signed).
  • Bitwise | ^ << >> (signed) — bitwise.si:6-32 (>> → i32.shr_s).
  • || short-circuit via ir_makeIf; @and/@or/@not keyword forms — logic.si:9-35.
  • ++ string concat → IRCall to the $str_concat WAT primitive — strings.si:4-14.

Control flow (fully data-driven). @if/@loop/@break/@continue/@return/@defer/@match each register an expression_keyword + handler — if.si:7-20, loop.si:6-20, control.si:6-37, match.si:5-10. @match delegates the variant-dispatch engine to the host helper compiler_expandMatchChain (intended CompilerAPI delegation, like the struct/sum expanders). @return routes through compiler_emitReturn so @defer cleanups run LIFO.

Casts. @toFloat/@toInt(+:Int64 typed overload)/@toInt64/@toU32/@toU64(+:Int64) — cast.si:9-86, using register::typed_keyword for the typed forms (the keyword analogue of typed_operator).

Definition kinds. @global/@fn (shared LetOrFn_lower), @local, @extern, @type_alias/@type_distinct (emit ir_null — typechecker-only), @type_sum/@enum/@type/@struct — defkinds.si:9-115, struct.si:14-20. The sum/struct/record keywords delegate layout + constructor synthesis to TS expanders (defExpanders.ts sumTypeExpander:52, typeRecordExpander:118, structExpander:285) — surface is data-driven, heavy lifting is documented host delegation.

Metadata. @platform (→ ir_null) and @export (→ ir_makeExport) — metadata.si:7-24.

StrataType coverage: Operator, Control, Metadata are the three fully-shipped data-driven StrataTypes (strataenum.ts, docs/strata.md:19-26).

Stdlib library code (the third, non-binary category). This is *not* strata and *not* primitive — it is ordinary Silicon source in compiler/src/stdlib/*.si that calls WASM:: intrinsics (which *are* strata operators) and the WAT primitives: mem, heap, num, str, io, slice/str_bytes, strbuilder, vec, hashmap, rc, option, result, iter. The Rc smart pointer (rc.si), the gc-mode identity shadows (gc/rc.si), the WASI preview1 @extern declarations (strata/modules/wasi_snapshot_preview1.si), and the @extern out-pointer convention (over $scratch_alloc) all correctly live as .si data. *(Note: "via-strata" in the dataset for these means "data-driven .si source," not "defines an operator/keyword stratum.")*

Platform/env module surfaces. web::/console::/JSString::/wasi_snapshot_preview1::/compiler:: are @extern host-import .si declarations in strata/modules/ loaded into a flat ModuleRegistry (loader.ts). The compiler:: surface (compiler.si, ~120 externs) is the comptime API that strata handlers call.

2. Should be primitives in the compiler — and why (the irreducible substrate + the "should-migrate" tension)

The irreducible substrate (must stay primitive — circular otherwise). These cannot be strata because strata are defined *in terms of* them:

  • LL(1) parser + lexer — parser/handwritten/parser.ts (class at :52, flat operator chain at :541), lexer.ts (maximal-munch at :126). The fixed grammar strata are parsed by; CLAUDE.md forbids grammar changes. New keywords need no grammar growth.
  • Comptime engine — comptime/engine.ts (compileHandlerToWasm:100, __compilingHandler:236, fixpoint at :292). The machine that turns .si data into running lowering logic; cannot itself be a stratum (chicken-and-egg, broken by the __compilingHandler flag).
  • CompilerAPI bridge — compiler-api/index.ts (createCompilerAPI:366). The IR-builder/AST/type/module/diagnostic interface strata bodies dispatch into.
  • WASM::*/IR::* intrinsic alphabet — intrinsics/intrinsics.ts (record at :21, resolveIntrinsicWasmInstr:188). The leaf ops strata reference by name; the boundary between .si and the byte emitter.
  • WASM byte emitter + LEB128 + opcode table — codegen/wasm-emitter.ts (OPCODES:109), codegen/leb.ts. The codegen floor.
  • Literal lowering (42, 3.14, '...') — lower.ts:1011-1023, allocString:156. The leaves lowerExpr bottoms out into; no AST node below them.
  • WAT runtime primitives — codegen/std.wat: $alloc bump allocator (:46-87), $alloc_string/$alloc_array/$scratch_alloc (:96-144), $mem_copy/$realloc (:152-184), $arena_promote (:199-203), $str_ptr/$str_len (:212-216), $heap_get/$heap_set/$heap_used/$arena_used (:227-248), $str_concat (:326-371). These manipulate the heap global, bulk-memory instructions, and host imports — not self-hostable in portable .si. Architectural caveat: std.wat is mirrored byte-equal in codegen/prelude-ir.ts (buildPrelude) for the direct binary emitter; any primitive change must be made in *both* (tests assert byte-equality).
  • Memory/codegen ABI plumbing — injectRefSlots/injectJSStringRefSlots/injectCharCodeArrayRefSlots (lower.ts:504-648), gc-vec.ts (WasmGC struct.new/array.new for Vec[Int]), externref valtype emission. These are target-ABI binary-encoding concerns, not language semantics. gc-vec.ts is the one stdlib data structure that *must* be a TS primitive because portable .si cannot express WasmGC array ops (gc/vec.si is an empty stub).
  • Component assembler — modules/component.ts (assembleComponent:154). A whole-program pre-parse source transform; cannot be data-driven .si.

The "should-migrate" tension — primitives that are surfaces-without-handlers. These register a strata surface but lower in TS; they are the Codegen-StrataType gap made concrete:

  • @try (lowerTry:1670) — the lowest-hanging fruit. It needs a fresh local + multi-statement Block, and those primitives (ctx.freshIdCounter, ctx.pendingLocals.push, ir.makeBlock) now exist on the API. A try.si stratum could own it.
  • @with_arena / @move_to_parent_arena (lowerWithArena:1885, lowerMoveToParentArena:2023) — harder: needs AST tail-walking (isMoveToParentArenaCall), per-target branching (wasm-gc elision), and a type-driven byte-size helper. Migrate together when the CompilerAst surface grows a tail-walk + type-size API.
  • @fnref / @call_indirect (lowerFnRef:1589, lowerCallIndirect:1625) — blocked on the funcref table being a LowerCtx field *absent from CompilerCtx*. Both are hardwired to a single __fn_i_i (i32→i32, no env) signature for v1.0.
  • Struct field read/write (lowerNamespace/lowerStructFieldChain:1072, lowerStructFieldStore:1174) — load-bearing TS because . field-access has no on::lower hook and the grammar parses x = v as BinaryOp('='). Needs a field-access dispatch surface.
  • Array-literal $[...] (lowerArrayLiteral:2199 + wasm-emitter.ts sentinel recognition) — a code smell: lowering is split across lower.ts and the emitter via a magic ARRAY_LITERAL_CALLEE string. Needs a real IRArrayLiteral node kind.
  • Legacy typed-operator intrinsic path (lower.ts:1313-1356) — Float/Int64/unsigned overloads that still resolve via stratum.data.intrinsic rather than on::lower; migrating them deletes ~40 lines of per-type TS and the !intrinsic precedence gate.

The biggest divergence: the QBE native backend (codegen/qbe/lower.ts). It lowers the type-checked AST straight to QBE IR text without consuming strata at all — its ElaboratorRegistry param is unused (:193) and it has its own lookupOpToQbe table (:548). Every operator/keyword the strata define must be *re-implemented* in TS for native builds: a second source of truth that can drift from operators.si. It still throws on @with_arena (:632). This is the strongest case for a real Codegen StrataType (one .si declaration carrying both WASM and QBE lowering).

Dead code to remove (now that migration shipped): the IR::control_or/control_and short-circuit branch in lowerBinaryOp:1326-1335 (Or_lower owns || now; the logic.si:3-5 comment is stale), the empty expander-map dispatch in lowerBuiltinCall:1535, and the corresponding control_* intrinsic entries — all dead for built-ins pending a grep for user strata.

3. Not implemented but should be — the gaps, prioritized (v1.0 gates vs post-bootstrap)

The one HARD v1.0 product gate: 100% web/bun FFI coverage, delivered by the ADR 0017→0018→0019 chain — all three are Proposed, "no code lands until Accepted." Today only scalars, linear-String pointers, JSString-as-externref, and CharCodeArray cross the boundary (~10%).

  • FFI bindgen (ADR 0017) — compiler/bindgen/ does not exist; ~54 host bindings are hand-written in three drifting places. Establishes the ~10% baseline + anti-drift lockstep.
  • Async/Promise FFI (ADR 0018) — no object handles, no @await, no Asyncify pass. Lifts coverage ~10%→80% on the Bun-portable baseline (JSPI is an off-path V8/SpiderMonkey-only fast path; Bun lacks it).
  • First-class closures (ADR 0019) — *the keystone*. No capture analysis, no $Clo struct, no Fn[] surface type. Named "the largest single item" and "the gate does not close without it." The mode-tiered design (escaping closures wasm-gc-only, engine-GC'd) dissolved the old free-list dependency; the runtime foundation is already verified (Bun runs wasm-gc + externref; v1.0 reuses existing call_indirect).

Post-bootstrap reserved StrataTypes (4 of 9 unbuilt):

  • Constraint — dead enum slot (strataenum.ts:14); operators dispatch via compound keys, not a Constraint node. Scope narrowly (Num/Comparable bounds, per ADR-0021 draft), never general typeclasses.
  • Runtime — no enum variant; the natural home for GMM pluggable allocators/schedulers.
  • Capability — no enum variant, no required_caps plumbing anywhere in src/. The security gate for the plugins/sandboxing niche.
  • DSL — no enum variant, no parse_dsl_region hook. The most advanced surface; reserve the hook now (bootstrap-plan already does) so it's non-breaking later.
  • Codegen — partial: control/def-kind lowering moved to .si, but new IR node kinds still require TS; ir/nodes.ts is a *closed* discriminated union, violating bootstrap-plan R1 (open-tagged IR). Must be honored during the self-host port or "strata as mods" needs a compiler rewrite.

The capability/borrow/effect cluster (ADRs 0011/0012/0013/0015) — designed, zero code. No borrow checker, no rcaps (&/&mut/&uniq/&val), no effect lattice, no on::check whole-program pass, no ocaps. All gated on a common substrate (on::check + AST reflection) that does not exist (only per-node hooks + the P0 comptime substrate ship). ADR 0011 calls the borrow checker "extremely low priority"; v1.x/v1.y ship without it. This is what makes positioning.md's "gradual safety" real rather than aspirational. The Span/View/Slice rename (ADR-0011 addendum) is deliberately deferred *because* there is no enforcement — today's Slice[u8] (slice.si) is the working stand-in.

Stdlib / iteration tails (v1.1): IterStep[T,R] protocol *dispatch* (type ships in iter.si, but @loop doesn't select an arbitrary next-bearing type — gated on reflection ∧ closures ∧ an IRLoop result slot ∧ heap-in-sum); generic Vec[T]/HashMap[K,V] monomorphization (today i32-specialized); free-list allocator (bump leaks until arena exit — $realloc "old block leaks"); GC-as-a-stratum (ADR 0008 M-6, the true gradual-MM payoff). The v1 iteration subset (a..b ranges, indexed Vec) *did* ship (ADR 0016, v0.1.3) as a syntactic desugar.

Module-system gaps (mostly correctly deferred): --emit=wit is over-claimed by ADR-0024 as "available at v1.0" — in reality only [package].namespace metadata is parsed (sigil_cli.ts:129-132); there is no flag, no emitter, no Silicon→WIT type table (rootExports are computed but never serialized). --emit=component, 3-segment ml::sub::fn dep calls, internal/ tier, nestable modules, binary-level dep merge, and sgl.lock pinning are all correctly absent per ADR.

Self-hosting — completely unstarted as code. The compiler is 100% TypeScript; there is no Silicon-source compiler dir, and the QBE backend throws on @with_arena (qbe/lower.ts:632). The qbe-self-host-plan.md is a 2026-06-01 "proposal." Inconsistency to flag: compiler/CHANGELOG.md "1.0.0 … Self-host verified" contradicts the code state and should be treated as aspirational.

Deliberate non-goals (do NOT add) — well-disciplined (ADR-0023): nominal traits/typeclasses (#3), methods/UFCS (#4), operator precedence (#5, *yanked in v0.1.4*), let-generalization/rank-N/row-poly (#6), GC-by-default, postfix operators, effect rows, algebraic-effect handlers, and CM async (the path not taken, ADR-0025). These define what Silicon *is not*.

Key Tensions & Recommendations

1. The Codegen StrataType gap is the central architectural tension. Strata register surfaces but lowering is still TS-hardcoded for @try/@with_arena/@fnref/@call_indirect/field-access/array-literals. This is not failure — it is the migration tail, blocked on specific missing CompilerAPI capabilities (funcref-table mutation on CompilerCtx, AST tail-walking, a field-access on::lower hook). Recommendation: migrate @try first (its blockers are gone), then close the funcref pair, then tackle the arena pair. Crucially, the open-tagged-IR requirement (bootstrap-plan R1) must be honored during the self-host port — ir/nodes.ts is a closed union today, and "strata as mods" cannot ship new IR node kinds without it.

2. The QBE backend is a second source of truth. It re-implements every operator/keyword in TS, divorced from operators.si, and can silently drift. Recommendation: a real Codegen StrataType where one .si declaration carries both WASM and QBE lowering; until then, treat lookupOpToQbe (qbe/lower.ts:548) as a known drift hazard and gate native builds behind oracle parity tests.

3. The capability model is the identity-vs-reality gap. Silicon's pitch is "safety from capabilities/effects, not GC/purity" — yet there is zero capability-gate machinery in the compiler. All four ADRs (0011/0012/0013/0015) are Proposed with "Implementation pointer: None." Recommendation: this is correctly low-priority for v1.0, but on::check + AST reflection are the shared substrate for *both* the optimizer (0012) and the checker (0013/0015) — build those two first when the work starts, since they're reused three ways.

4. FFI closures are the v1.0 keystone, and the gate is entirely unbuilt. The single non-negotiable v1.0 deliverable (100% web/bun FFI coverage) depends on a three-link chain where nothing has landed. Recommendation: sequence strictly — bindgen → object-handles + Asyncify (Bun-portable) → closures C0–C3 → poll-reactor. Closures are highest-risk/highest-effort; start the escape classifier design now.

5. The module system is a source-merge front-end, not a linker. assembleComponent (component.ts:154) statically merges all modules into one flat Silicon string before the unchanged compile() runs; mod::f is resolved to mod__f by scope-aware text rewrite (walkScoped:654-759), never touching ModuleRegistry. This is a *clean divergence* from ADR-0024's "registry precedence layer" wording — arguably better (zero registry changes, web::/WASM:: untouched). Recommendation: keep it; document the divergence; fix --emit=wit over-claim in the ADR; thread per-file spans into E-DUP-DEF (currently reports m.dir, not both file:line).

6. Doc-vs-code drift on StrataTypes. docs/strata.md names 9 StrataTypes; the code enum has 6 differently-named variants. The spec's "Type" is the code's "Definition"; "Codegen" is a deferred comment. Recommendation: reconcile the table with the enum so the mod-surface roadmap is legible.

7. Stale in-source comments outran the implementation — twice. The logic.si:3-5 "|| keeps its hardcoded short-circuit" comment is stale (Or_lower owns it). The result.si:13 "@try deferred" comment is stale (@try is fully shipped, dispatched, and has 5 passing tests in try.test.ts). Recommendation: cross-check deferred/TODO comments against the test suite and strata wiring before trusting them; the @try entry in particular should be re-graded implemented (keyword stratum + TS-hardcoded lowerTry), with only the ? postfix operator genuinely missing.

Prioritized Roadmap

v1.0 GATES (the hard, non-negotiable path — build in this order):

  • FFI bindgen (ADR 0017) — compiler/bindgen/ + bindgen.lock + CI check-shims. Establishes the ~10% baseline and stops the three-way hand-binding drift. Independently buildable.
  • Async/Promise FFI P0/P1 (ADR 0018) — object handles + Asyncify @await on the Bun-portable baseline. Lifts ~10%→80%. (JSPI P2 is off-path optimization.)
  • First-class closures C0–C3 (ADR 0019) — generalize the __fn_i_i ABI → non-escaping all-modes → escaping wasm-gc-only + the escape classifier (the gate) → poll-reactor. Highest risk, highest leverage; the gate does not close without it. Foundation already verified present (Bun + wasm-gc + externref + call_indirect).

HIGH-VALUE NON-GATE (cheap, unblocks ergonomics & cleans the architecture):

  • Migrate @try to a try.si stratum — all required primitives now exist; removes a hardcoded keyword from lowerBuiltinCall and is the proof-of-concept for the Codegen StrataType.
  • Delete dead code — the control_or/control_and branch (lower.ts:1326-1335), the empty expander-map dispatch (:1535), and the legacy typed-operator intrinsic path (:1313-1356); fix the stale logic.si/result.si comments.
  • Open-tag the IR node table (bootstrap-plan R1) — do this *as part of* the self-host port, not after; otherwise the Codegen StrataType / "strata as mods" needs a full rewrite.
  • Fix the --emit=wit over-claim — either ship a minimal world-skeleton emitter from rootExports over the scalar subset, or correct ADR-0024's Implementation-status to match reality.

POST-BOOTSTRAP / STRATEGIC (deliberately deferred, v1.1+):

  • Self-hosting (QBE → Silicon-in-Silicon) — extend the QBE emitter to WASM-parity (it throws on @with_arena at qbe/lower.ts:632), then port the compiler; oracle-gated. Unblocks Strata-as-Mods and the licensing/binary-size win (drop ~88MB LGPL Bun → MIT native ELF). Correct the CHANGELOG's premature "Self-host verified."
  • Generic container monomorphization — Vec[T]/HashMap[K,V] beyond i32; the "dominant speed lever" and a prerequisite for iterables dispatch.
  • The capability cluster (ADRs 0011/0012/0013/0015) — on::check + AST reflection first (reused by optimizer and checker), then rcaps/borrow checker, then ocaps; makes "gradual safety" real. Bundle the Span/View/Slice rename here.
  • IterStep protocol dispatch (ADR 0016 tail) — gated on closures ∧ reflection ∧ loop-result-slot ∧ heap-in-sum; record now so all four workstreams aim at the same protocol.
  • Reserved StrataTypes (Constraint/Runtime/Capability/DSL) — post-self-host mod surfaces; Capability is most strategic (plugins/sandboxing), DSL the latest. Free-list allocator + GC-as-a-stratum (ADR 0008 M-6) land here as the gradual-MM north-star payoff.

DO NOT BUILD (settled non-goals, ADR-0023): traits/typeclasses, methods/UFCS, operator precedence (yanked once already), let-generalization, GC-by-default, postfix operators, effect rows, algebraic-effect handlers, CM async.
